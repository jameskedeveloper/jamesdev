How It All Started (2019 - Present)

My coding adventure began in 2019 with a second-hand laptop and a lot of curiosity. I learned the basics — HTML and CSS — by building small pages and trying out simple designs. Those first projects were imperfect but gave me that addictive spark: seeing code transform into something visible and useful.

In 2020, I moved on to JavaScript and basic backend concepts. I spent nights debugging, following tutorials, and rebuilding the same small apps until they worked. It was slow progress, but each deployed project and piece of feedback taught me practical problem-solving and the importance of persistence.

By 2021, I was tackling more complex builds — interactive sites, simple APIs, and automation scripts. I started helping friends and local small projects, which pushed me to write cleaner code, consider user needs, and manage projects from concept to deployment. Collaboration and real users accelerated my growth.

Looking back, the journey was never just about learning languages or frameworks. It became a way of thinking: break problems down, iterate fast, and learn from mistakes. That mindset is what keeps me building today — always improving, always experimenting.

Creating Tools (2022 - Today)

In 2022 I shifted from learning to creating — focusing on small, practical tools that solve real problems. I began with automation scripts to remove repetitive tasks, and quickly expanded into messaging bots and utilities that make day-to-day tasks faster and simpler.

Over the next months I built chatbots, messaging automation (including WhatsApp tooling), and features such as media downloaders, music play commands, and auto-status utilities. Each tool taught me new technical lessons: API integration, file handling, rate limits, and how to design commands that are reliable in real environments.

I also made a conscious choice to ship real, working functionality — no placeholders. That meant building end-to-end experiences, handling edge cases, and listening closely to user feedback. Improving UX, keeping commands predictable, and ensuring the tools actually help people became core priorities.

Today these tools continue to evolve — I'm iterating on performance, adding safeguards, and planning features that bring real value. The goal is the same: build straightforward, reliable tools that save time and make technology more accessible.
